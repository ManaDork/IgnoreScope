# Backlog

Items parked here are ideas or features that aren't ready for active work.

## When to park an item
- The idea needs more thought before specification
- Dependencies aren't ready yet
- Lower priority than current work
- Blocked by external factors

## When to pull an item
- Dependencies are resolved
- It becomes relevant to active work
- Priority shifts make it timely

## Format
One markdown file per backlog item with:
- **Summary**: What and why (1-2 sentences)
- **Blocked by**: What needs to happen first (if anything)
- **Estimated scope**: S / M / L
